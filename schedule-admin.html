ניהול זמינות לפגישות

דף פרטי – אל תשתפי קישור. לחצי שמור – עדכון ל-GitHub (אם מולאו פרטים) או העתקת JSON ללוח.

כניסה

סנכרון עם דף „קביעת פגישה”: הדף הציבורי קורא את naomi-availability.json מהאתר. אם ה-fetch נכשל, נטען גיבוי מתוך index.html (תג naomi-availability-embed) – חייב להיות זהה לקובץ.

GitHub Pages – אחרי כל שינוי בלוח:
  1. לחצי שמור – עם פרטי GitHub מלאים הקובץ יעלה לריפו (אם הדפדפן מאפשר). בלי פרטים – יועתק JSON ללוח להדבקה ב־index.html / בעריכה ב-GitHub.
  2. ב־index.html עדכני את כל התוכן בשורה של <script type="application/json" id="naomi-availability-embed"> (שורה אחת, אותו JSON כמו בקובץ).
  3. אם ערכת ידנית: git add → commit → push לענף שממנו נבנה האתר (לרוב main).
  4. המתיני דקה לפריסה, רענון קשיח (Ctrl+F5).
מדריך מפורט: github-schedule-sync.md בתיקיית הפרויקט.

יומן זמינות

ימי שבוע: בכרטיס „ימים בשבוע לקביעה” למטה בוחרים אילו ימים בשבוע בכלל פתוחים בדף הפגישות – השאר יופיעו שם כלא זמינים. בלוח: לחיצה על תאריך עתידי (ביום שבוע פתוח) = חסימה נוספת לאותו תאריך בלבד (אדום).

פתוח לקביעה חסום

חודשים שעברו באפור. ימי שבוע שלא סימנת בכרטיס למטה – מוצגים כאן באפור (לא ניתן לחסום בהם תאריך ספציפי דרך הלוח). בסוף: שמור + ודאי שהקובץ/ההטמעה באתר עודכנו.

ימים בשבוע לקביעה

רק הימים המסומנים יהיו ניתנים לבחירה בדף קביעת הפגישה. ימים לא מסומנים = תמיד לא פנויים שם (בכל התאריכים). זה נשמר בשדה daysOpen בקובץ ה-JSON.

שעות

שעת התחלה ושעת סיום כוללות דקות (למשל 09:30–18:45). סוף החלון = המועד האחרון שבו פגישה יכולה להסתיים.

הגדרות סנכרון GitHub (מתקדם)

מלאי פרטי המאגר ו-Personal Access Token (GitHub → Settings → Developer settings → Fine-grained token: הרשאת Contents read+write על הריפו).
כשהשדות מלאים, לחיצה על שמור בראש הדף תדחוף את naomi-availability.json לריפו. אם הדפדפן חוסם (CORS) – „דחיפה ידנית”, „פתיחת עריכה” או tools/push-availability.mjs.
